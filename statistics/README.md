# statistics/

Product analytics, metrics, and observability planning. **No real data yet (pre-launch).**

## Structure
- `instrumentation-plan.md` — what to measure + the privacy decision (read first).
- `metrics-glossary.md` — KPI definitions.
- `dashboards/` — dashboard specs/exports (later).
- `reports/` — periodic performance reports (post-launch).

The app currently ships **no telemetry** and is positioned as offline/no-tracking. Measurement
planning should focus on hook health first: onboarding completion, first workout, ritual return,
collection engagement, and whether the user keeps strengthening their character. See
[CLAUDE.md](CLAUDE.md).
